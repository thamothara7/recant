# Week 2 plan: taint engine + quarantine transaction (2026-07-03)

Scope per spec sections 5 and 12: **taint-engine** (explicit closure via recursive
CTE, implicit closure via vector kNN in a contamination window, candidate set with
scores), **quarantine-service** (`recant(source_id)` as one serializable
transaction), tests including quarantine atomicity under concurrency. Plus the two
items deferred from W1: deterministic fake embedder and structured JSON logging
with `incident_id` correlation. Everything runs against the local chaos cluster;
nothing here needs U1/U3.

## 1. Taint closure algorithm

Seeds: `SELECT belief_id FROM beliefs WHERE source_id = $recanted`.

Two expansion operators, iterated to a fixpoint inside one transaction:

**Explicit expansion** — recursive CTE over `derivations`, kind-agnostic (explicit
edges from the write path AND inferred edges materialized by earlier recants both
propagate taint):

```sql
WITH RECURSIVE tainted (belief_id) AS (
    SELECT belief_id FROM beliefs WHERE belief_id = ANY(%s)   -- frontier
    UNION
    SELECT d.child_id FROM derivations d JOIN tainted t ON d.parent_id = t.belief_id
)
SELECT belief_id FROM tainted;
```

**Implicit expansion (vector kNN)** — for each closure member that has an
embedding, a top-K query ordered by cosine distance:

```sql
SELECT belief_id, embedding <=> %s::vector AS dist
FROM beliefs
ORDER BY embedding <=> %s::vector
LIMIT %s;   -- K, default 20
```

Verified empirically on the local v26.2.3 cluster (2026-07-03): the default
`CREATE VECTOR INDEX` from migration 0002 serves only L2 `<->` ordering; a
cosine `<=>` ORDER BY full-scans. With `vector_cosine_ops` the same query plans
as `vector search`. **Migration 0004 rebuilds `beliefs_embedding_idx` with
`vector_cosine_ops`.** No index hint needed once the opclass matches; the
integration suite asserts the `vector search` plan node via EXPLAIN so a
regression to full scan fails tests (the judge overlay must never claim an
index-backed kNN that is actually a scan).

Filters applied in the application after the top-K (keeping the query in the
index-friendly ORDER BY + LIMIT shape):
- similarity `1 - dist >= TAINT_THRESHOLD` (env `RECANT_TAINT_THRESHOLD`, default 0.80)
- `created_at >= window_start` where `window_start = min(created_at)` over the
  explicit seeds — contamination cannot flow backward in time
- not already in the closure; status is `active` or `suspect` (never resurrect
  `retracted`); embedding not null

Each implicit hit is recorded as a candidate inferred edge
`(child = hit, parent = the closure member it matched, kind='inferred', score=similarity)`.

**Fixpoint loop**: alternate explicit CTE (seeded from the new frontier) and kNN
expansion until a round adds nothing. Hard cap 10 rounds (demo graphs are tiny;
the cap is a runaway guard, logged if hit).

The engine is a library function `compute_closure(conn, source_id, ...) -> Closure`
that runs on a caller-provided connection so the quarantine service computes it
INSIDE its serializable transaction (no TOCTOU between preview and flip). A
read-only preview endpoint exposes the same function for the console's dialog.

## 2. Quarantine transaction (`recant(source_id)`)

One `run_txn` serializable transaction, retried on 40001:

1. `compute_closure(conn, source_id)`.
2. Materialize the candidate inferred edges:
   `INSERT INTO derivations ... ON CONFLICT (child_id, parent_id) DO NOTHING`.
3. Flip: `UPDATE beliefs SET status='quarantined' WHERE belief_id = ANY($closure)
   AND status IN ('active','suspect') RETURNING belief_id, agent_id` — the RETURNING
   set is the *newly flipped* count (idempotent re-recant flips 0).
4. `INSERT INTO incidents (source_id, opened_by) RETURNING incident_id` — every
   recant call is a new incident row; the audit trail records repeat calls honestly.
5. Attested action: `sig = Ed25519(dev_signer(actor), SHA256(canonical_action_payload))`
   over `{incident_id, source_id, sorted(newly_flipped_ids), belief_count, ts}` using
   the same canonical JSON encoding as the belief chain (sort_keys, compact).
   `INSERT INTO quarantine_actions (incident_id, belief_count, actor, sig)`.
6. Outbox event for the W3 changefeed/fanout:
   `INSERT INTO memory_events (kind='recant', incident_id, payload)` with payload
   `{source_id, belief_ids (full closure), newly_flipped_ids, agent_ids, actor}` —
   one row per recant; the fanout Lambda expands it into per-agent evictions.

Concurrency semantics (corrected by the 2026-07-03 design review; documented, tested):
- A concurrent reader sees the flip all-or-nothing (serializable isolation). The
  atomicity test makes this deterministic with a test-only hook that parks the
  recant transaction between the flip and commit while a reader proves the old
  state is still what it observes.
- The real guarantee for concurrent writes is timestamp-based, not "one side
  retries": the closure is complete as of the recant's commit timestamp. A
  derivation committed below that timestamp is included; one committed above it
  is post-recant residue. Residue is handled on the write path: the gateway now
  marks a new belief `suspect` at birth when it cites a source with an open
  incident or derives from a `suspect`/`quarantined` parent. This is also the
  producer for the `suspect` status (previously a dead enum state); runtime
  eviction of residue remains the W3 changefeed's job.
- TTL interplay: TTL deletes are ordinary DELETEs and the derivations FKs
  (NO ACTION) block them, so expired beliefs persist and the TTL job errors —
  recorded in the README failure-modes table, deliberately kept (provenance
  edges must never be severed). `RECANT_UNTRUSTED_TTL_DAYS` makes the window
  configurable; the deployed demo sets 90 so nothing expires mid-judging.
  Expired-but-visible beliefs still join the closure and flip (pinned by test).
- kNN recall: filters run after the top-K, so the engine widens K adaptively
  (double, bounded by RECANT_TAINT_MAX_K=320) while the farthest returned
  neighbor still clears the threshold; a still-hot boundary at the cap sets
  `capped` and logs. ANN beam-size tuning at the W6 seed scale is a W6 item.

## 3. Embedder (deferred W1 item)

`services/common/embedder.py`: `Embedder` protocol with `embed(text) -> list[float]`
(1024-dim, unit-norm). W2 ships `HashEmbedder` — token-level: each lowercased
alphanumeric token hashes (SHA256-seeded) to a deterministic unit vector; the text
embedding is the normalized token-vector sum. Properties: deterministic across
runs, similarity is real token overlap (paraphrases sharing vocabulary score high,
unrelated text near 0) — an honest miniature of an embedding model, not a rigged
lookup. Bedrock Titan lands behind the same protocol in W3/W4.

Tests that need exact threshold control construct explicit vectors (basis vectors
and mixtures) and pass them through the gateway's existing `embedding` field.

Measured 2026-07-03: the console-fixture paraphrase pair scores 0.109 under
HashEmbedder (no shared tokens after tokenization), while a natural shared-vocab
paraphrase scores 0.60 and unrelated content stays under 0.11. Thresholds are
therefore a property of the embedder: `HashEmbedder.default_threshold = 0.35`
(lexical scale, comfortable margin both ways), Titan's stays 0.80 when it lands
in W3. `RECANT_TAINT_THRESHOLD` overrides either. The W2 seed story uses
paraphrase text with honest vocabulary overlap; the divergent-wording version of
proof moment 2 arrives with Titan.

- `GET /healthz`.

Not in W2 (logged here so it is a decision, not an oversight): the gateway does
not yet reject new writes citing a recanted source or quarantined parents —
eviction of post-recant residue is W3's changefeed job. Revisit in W3.

## 6. Tests

Unit: HashEmbedder determinism + norm + overlap ordering; canonical action payload
stability; closure round-cap guard.

Integration (live cluster, existing conftest):
- explicit closure: recant flips the seed belief and its explicit descendants
- implicit closure: paraphrase with similar embedding and NO explicit edge flips;
  inferred edge materialized with score
- transitivity through inferred: explicit child of the paraphrase also flips
- threshold: dissimilar embedding untouched
- window: similar belief created BEFORE contamination entered is untouched
- retracted beliefs are not resurrected to quarantined
- idempotency: second recant flips 0, writes a second incident + action
- attestation: quarantine_action sig verifies against the actor's pubkey
- outbox: memory_events payload complete and correct
- atomicity: concurrent readers polling during recant never observe a partial flip
- 404 on unknown source

## 8. Post-implementation review adjustments (2026-07-04)

A 4-lens adversarial review after the code landed produced these changes,
recorded in full in `docs/plan.md` decisions 14-16:

- **Attested action is self-verifying.** The signed payload binds
  `newly_flipped_ids`, which had lived only in the unsigned outbox event.
  Migration 0005 adds `quarantine_actions.newly_flipped_ids`; a forensics
  verifier holding only the database recomputes the canonical payload from
  `quarantine_actions` + `incidents`.
- **Action keys domain-separated.** Actions sign with `dev_action_signer_for`
  (`recant-dev-action:{actor}`), disjoint from agent belief-chain keys, so an
  unauthenticated `actor` string cannot forge a signature attributable to an
  agent. KMS-ARN registry pinning is W4.
- **One clock domain.** The gateway stamps `beliefs.created_at` from the DB
  clock (`SELECT now()` inside the txn) to match `sources.created_at`; the
  window filter no longer depends on host/DB clock agreement.
- **Two incompleteness signals.** `Closure.capped` split into `rounds_capped`
  and `knn_truncated`, logged distinctly and surfaced in `ClosureOut`.
  Inferred-edge parent is the highest-similarity probe per hit, not scan order.
- **Atomicity test hardened.** The reader runs `PRIORITY HIGH` and the test
  asserts it completed pre-commit, so determinism no longer rides on
  `kv.transaction.write_buffering`.
- **Deferred:** a retracted kNN hit still halts implicit traversal (asymmetric
  with the explicit path). Not reachable in W2 (no retract API); revisit when
  one ships. The "self-poisoning seed" finding was stale — the controlled-vector
  seed and `check_story`'s negative assertion already prevent it.
